HarborCrest: Visual Analytics Investigation

Was the embargo breach deliberate — or did the system simply break down under pressure?
912 communications 23 sessions 7 AI agents 6 channels May 17 – June 5, 2046
Loading dataset...

On June 5, 2046, TenantThread's automated communications system published confidential merger details on the social media platform FleX — one hour before the agreed 6:00 PM disclosure deadline. The acquiring company, CivicLoom Realty Partners, has engaged this investigation to answer a single legal question: was the embargo breach deliberate, or did the system simply break down under pressure?

This tool presents two weeks of internal communications from TenantThread's AI-driven platform — seven agents coordinating across six channels, producing 912 messages across 23 sessions. What makes this dataset unusual is that every message includes an internal record of the agent's reasoning: what it was reacting to, how it rationalized its decision, and what it was deliberating before acting.

The investigation is structured across three steps. Step 1 reconstructs the sequence of events via an annotated timeline. Step 2 establishes a behavioral baseline, contrasting how each agent operated during the quiet weeks of May against how it behaved on June 5. Step 3 searches the full record for leading indicators — earlier moments where the system was already behaving outside expected parameters. Selecting any round opens the Round Evidence Panel at the bottom of the page, where you can read agent reasoning, out-of-role post content, and the external briefing agents received that session.

Step 1 Event Timeline — Key Events Leading to the Embargo Breach

The timeline shows all 23 sessions ordered chronologically on a non-uniform scale: the 13 daily pre-crisis sessions are compressed on the left while the 10 hourly crisis-day sessions are expanded on the right, making the shift in system tempo visually explicit. Each circle on the spine represents one session; color encodes severity (grey = routine, amber = pressure, red = critical, dark red = breach). Stems rising above the spine connect to a labeled event box describing what happened internally that session. Stems dropping below the spine connect to a box showing how many public posts went out on FleX that session, when any did.

How to use this view:

Click an event box or circle on the spine (the labeled box above the spine) — brings the box to the front, so you can read its full label without the surrounding boxes overlapping it. The boxes overlap by design to fit 23 sessions of history into one view; this lets you inspect any one of them clearly.

Click anywhere else on the page — collapses the highlighted box back down and un-dims all the others, returning the timeline to its default state.

Double-click an event box, or click the circle on the spine itself — selects that round. This opens the Round Evidence Panel at the bottom of the page, scrolls you down to it automatically, and shows the full agent reasoning and message content for that specific session.

Hover over any circle or event box — shows a tooltip with the exact timestamp, the event summary, and the number of out-of-role posts sent that session, if any.

Step 2 Behavioral Heatmap — Agent × Channel Message Counts

This heatmap compares how each of the seven agents used each of the six communication channels across the pre-crisis period (left panel, 13 sessions) and crisis day (right panel, 10 sessions). Color luminance encodes message count on a sequential blue ramp — darker means more activity. Three anomaly types are automatically detected and color-coded: red = out-of-role posting (channel unused in baseline, now active for an agent with no public role); amber = channel spike (≥3× increase, ≥10 messages); blue-grey = activity drop (≤50% of baseline, ≥10 message decrease).

How to use this view — every interaction, explained:

Hover over any cell — shows the exact message count for that agent and channel, the pre-crisis baseline for comparison, and the anomaly classification if one applies.

Select a round (by clicking a circle on the timeline, a bar in Step 3, or using the round selector at the top of the page) — every cell belonging to an agent who sent at least one message during that round gets a thick navy border. This does not change which period the heatmap is showing (pre-crisis vs. crisis) — it simply marks, on whichever period the selected round belongs to, which agents were actually active in that specific round. A navy-bordered cell means: this agent sent a message on this channel at some point during the selected round.

Step 3 Anomaly Bar Chart — Out-of-Role Posts per Round

Each bar represents one session, ordered chronologically. Bar height encodes the count of out-of-role public posts — anonymous posts by any agent, or personal posts by agents with no legitimate public role (Legal-Agent, Judge-Agent, Platform-Trust-Agent). The dashed red baseline at zero marks expected behavior: all 13 pre-crisis bars should sit flush against it. Any bar rising above it indicates a session where the system was already operating outside expected parameters. Color encodes deviation magnitude (blue = zero, amber = 1, red = 2–3, dark red = 4+).

How to use this view — every interaction, explained:

Hover over any bar — shows a tooltip listing exactly which agent made each out-of-role post that session, and on which channel, so you can see who was responsible for the deviation before clicking in further.

Click a bar — selects that round. This opens the Round Evidence Panel below, showing the full content of every out-of-role post, the internal reasoning of every active agent, and the environment context for that session.

Click the same bar again — deselects the round and returns the evidence panel to its placeholder state.
